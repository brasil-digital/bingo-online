02 — Stack Tecnológico
🗄️
Banco de Dados
Supabase · PostgreSQL
Realtime nativo via WebSocket. RLS para segurança por ponto. Supabase Edge Functions para lógica crítica do sorteio. Você já domina.
⚡
Realtime
Supabase Channels
Broadcast de cada bola sorteada para todos os displays simultaneamente. Latência <100ms. Sem WebSocket custom necessário.
📺
Display TV
React PWA · Vite
URL única por ponto. Abre em qualquer Smart TV com browser. Fullscreen automático. Sem app para instalar no bar.
📱
App Vendedor
Ionic/React · seu stack
O operador do bar vende cartelas, vê comissão em tempo real, confirma pagamentos PIX e acompanha o sorteio pelo celular.
⚙️
Motor de Sorteio
Node.js + Cron
Sorteia 1 bola a cada N segundos. Detecta Kuadra/Kina/Keno automaticamente. Seed auditável para transparência. Roda como microserviço.
💸
Pagamentos
Iugu · PIX API
PIX gerado automaticamente na venda da cartela. Webhook confirma pagamento e ativa cartela em tempo real. Repasse automático para pontos.
🔔
Automações
n8n · Z-API
WhatsApp "sorteio começa em 10 min". Notifica ganhador com comprovante. Relatório diário para operadores. Você já usa n8n.
📊
Admin / Analytics
React + Retool
Retool para dashboard interno rápido. React para painel white-label que você vende para operadores. Relatórios financeiros por ponto.

04 — Fluxo Operacional
1
Operador Central cria rodada
Define prêmios (Kuadra/Kina/Keno), preço da cartela, horário, pontos participantes e acumulado. Sistema gera cartelas em lote (até 240.000 por série).
2
Pontos vendem cartelas
Vendedor usa app Ionic, gera QR Code PIX. Comprador paga, webhook Iugu confirma, cartela ativa automaticamente. Opção de cartela física impressa (térmica).
3
n8n dispara notificações
"Sorteio começa em 15 minutos" via WhatsApp para todos os compradores do ponto. TV do bar recebe sinal de "pré-sorteio" e exibe contagem regressiva.
4
Motor sorteia em tempo real
Node.js sorteia 1 bola a cada 4s. Supabase Realtime faz broadcast para todos os displays simultaneamente. Algoritmo verifica todas as cartelas a cada bola.
5
Ganhador detectado automaticamente
Sistema identifica cartela ganhadora, exibe "GANHADOR" na TV com nome do ponto e números destacados. Push notification para o comprador. Relatório gerado.
6
Repasse financeiro
Iugu split automático: prêmio para ganhador via PIX, comissão para ponto, receita para operador. Relatório diário via n8n para cada parceiro.

06 — Vantagens sobre o Sistema Atual
❌ Sistema Legado (On Line Bingo)
  • Interface dos anos 2000, sem responsividade
  • Cartelas apenas físicas (papel impresso)
  • Sem PIX integrado — dinheiro no bar
  • Sem app para o jogador acompanhar
  • Sem notificações WhatsApp automáticas
  • Sem analytics para o operador
  • Código fechado, sem API
  • Servidor europeu com latência maior
✅ Bingo 2.0 (seu sistema)
  • UI moderna, animada, profissional
  • Cartela digital no celular do jogador
  • PIX automático integrado (Iugu)
  • Jogador acompanha pelo smartphone
  • WhatsApp automático via n8n + Z-API
  • Dashboard em tempo real por ponto
  • API aberta para integrações
  • Servidor brasileiro, latência mínima
